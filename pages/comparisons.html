Stemming vs Lemmatization

Morphological normalisation techniques for Marathi text

compare_arrows VS

content_cut Stemming

Definition

Heuristic process that chops off affixes to reduce words to a base root form without full linguistic knowledge.

Working Principle

Rule-based suffix stripping using predefined patterns (e.g., removing -णे, -तो, -ली, -ले from Marathi verbs).

Example

"खेळणे, खेळतो, खेळली" → stem: "खेळ"

Advantages
  • Fast & computationally cheap
  • Improves recall in IR
Limitations
  • Over-/under-stemming
  • Loses morphological info

dictionary Lemmatization

Definition

Linguistic process that reduces words to their canonical dictionary form (lemma) using vocabulary and morphological analysis.

Working Principle

Combines a Marathi morphological analyzer with a lexicon; considers POS tags and grammatical context to find the root.

Example

"गेलो, गेली, गेले" → lemma: "जा" (gender/number resolved)

Advantages
  • Linguistically valid lemmas
  • Superior precision
Limitations
  • Needs comprehensive lexicon
  • Slower than stemming
Parameter Stemming Lemmatization
Approach Heuristic suffix chopping Dictionary + POS-aware analysis
Output May produce non-dictionary stems Always a valid linguistic lemma
Speed Fast — no external resources Moderate — lexicon lookups
Marathi Suitability Limited — struggles with vibhakti system Excellent — handles inflectional richness
lightbulb
Verdict for Marathi NLP

Lemmatisation is strongly preferred over stemming for Marathi due to the language's rich inflectional morphology. Stemming's aggressive suffix removal often produces invalid roots, while lemmatisation preserves grammatical information essential for downstream tasks like NER and parsing.

TF vs TF-IDF

Term weighting schemes for feature extraction

compare_arrows VS

bar_chart TF (Term Frequency)

Definition

Measures how frequently a term appears in a document, normalised by document length.

Working Principle

TF(t,d) = count(t in d) / total terms in d. Common variants include log normalisation and augmented frequency.

Example

"मराठी भाषा मराठी संस्कृती" → TF("मराठी") = 2/4 = 0.5

Advantages
  • Length normalisation
  • Simple & interpretable
Limitations
  • Common words dominate
  • No cross-document discrimination

score TF-IDF

Definition

Product of Term Frequency and Inverse Document Frequency, weighting terms by local importance and global rarity.

Working Principle

TF-IDF(t,d) = TF(t,d) × log(N / df(t)). High weight when a term appears many times in few documents.

Example

"न्यायालय": TF=0.1, IDF=2.0 → TF-IDF=0.20 vs "आहे": TF=0.05, IDF=0.02 → TF-IDF=0.001

Advantages
  • Balances local & global importance
  • Reduces stopword influence
Limitations
  • No semantics / synonymy
  • Static, non-contextual weights
Parameter TF TF-IDF
Scope Local (within a document) Global (across the corpus)
Stopword handling Frequent words still score high Automatically down-weights them
Best for Single-document analysis Corpus-wide retrieval & ranking
Marathi Suitability Weak — common case markers dominate Strong — domain terms like "न्यायालय" gain weight
lightbulb
Verdict for Marathi NLP

TF-IDF consistently outperforms raw TF for Marathi text because it suppresses high-frequency case markers (like "चा", "ची", "ला") while boosting discriminative domain terminology — particularly valuable for legal and financial document retrieval where specific terms carry high evidentiary value.

Word2Vec vs FastText

Static word embedding architectures for Marathi

compare_arrows VS

network_intelligence Word2Vec

Definition

Predictive neural embedding model learning word vectors from local context windows via CBOW or Skip-gram architectures.

Working Principle

CBOW predicts target from context; Skip-gram predicts context from target. Uses negative sampling or hierarchical softmax.

Example

Skip-gram: given "मराठी", predict ["प्राचीन", "भाषा", "समृद्ध"]

Advantages
  • Efficient on large corpora
  • Captures linear analogies
Limitations
  • No OOV word handling
  • Ignores subword information

speed FastText

Definition

Extends Word2Vec by representing each word as a bag of character n-grams; the word vector is the sum of its subword vectors.

Working Principle

Each word split into character n-grams (3–6 chars); each n-gram gets a vector; word vector = sum of n-gram vectors. Trained via CBOW or Skip-gram.

Example

"मराठी" → n-grams: "<मर", "मरा", "राठ", "ाठी" ... → vector sum → embedding

Advantages
  • Embeds unseen inflected words
  • Ideal for Marathi morphology
Limitations
  • Higher memory for n-gram storage
  • Noisy vectors for short words
Parameter Word2Vec FastText
Granularity Whole-word level only Subword (character n-gram) level
OOV handling None — unknown words get no vector Strong — composes from subwords
Memory Lower — vocab-size vectors only Higher — n-gram table overhead
Marathi Suitability Poor — cannot embed inflected OOV forms Excellent — handles Marathi's rich morphology
lightbulb
Verdict for Marathi NLP

FastText is the clear winner for Marathi. Because Marathi is morphologically rich with complex inflectional paradigms, Word2Vec's whole-word approach fails on unseen inflected forms. FastText's subword n-gram representation can compose vectors for any inflected variant — critical for real-world Marathi NLP where the vocabulary is vast and open-ended.
